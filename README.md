<div style="margin-top: 20px">
　Oracle DBを題材に学習していこうと思っています。<br>
　使用する教材は以下になります。<br>
<ul>
    <li>
        <a href="https://livesql.oracle.com/landing/">Oracle Live SQL</a>

　前半はハンズオン形式で行って、余った時間はPing -tで知識の確認を行いたいと思います。<br>
　全4回か5回くらいを予定しています。正規化と連関エンティティなどの概念的な内容も、皆さんの理解度の状況を見ながらですが、扱かっていけたらなと思っています。<br>
　本当に基礎の基礎からやっていきます。単純なselect文から、left joinといった結合やテーブルの作成もやっていきます。<br>
　外部キーや参照動作のCASCADEなどの設定もできたら良いなと思いつつ、皆さんダルそうならやめとこうかなと思いつつな感じです。<br>
　あまり複雑なことやりすぎると付いて来れない方がでてきてしまうのではないかという懸念点があります。<br>
　DB側の参照動作については私は実務で扱ったことありませんが、他の方のお話を聞くと扱っているシステムもあるみたいです。私の扱ったことのあるシステムでは、DB側では制御せずアプリケーション側で全て制御していました。<br>
　聞いた話だとアプリケーション側でもDB側でも両方で行っているシステムもあるらしいです。私はややこしいのは苦手なので正直そのシステムは扱いたくないなと思ってしまいました笑<br>
　新人が入るたびに修正漏れが発生する！みたいな仕様のシステムは本当にPMさん大変だと思います。修正する側も大変ですが、管理する側はもっとストレス溜まるんだろうなぁと思います。(「またかよ！いいかげんにしろ！」みたいな...)<br>
　SQL文やストアドプロシージャのエラーについては、私が新卒で働いていた頃はかなり苦労した覚えがあります。ですが、今は生成AIに投げれば詳しく教えてくれるので、かなりDBの開発の難易度は下がったと思います。<br>
　DBとお友達になれるのかは、まず朝起きられるかに全てがかかっています。朝6:30までに目覚ましをかけたとして、起きたときに眠いけど11axがもし1人だったらかわいそうだから参加してやるかと思って起きられる方はDBともお友達になれるし、私ともお友達になれます。<br>
　寝坊した方や参加できなかった方は片思いにしかなれませんが、ToDoリストや実行したSQL文をアップロードしておくので、よかったら枕を濡らしながらご活用してください。<br>
　ご質問等あれば全体でメンション付けるかDMでも良いので、気軽にご連絡ください！
